3.1 USER REGISTRATION & EMAIL VERIFICATION FLOW
═════════════════════════════════════════════════

MERMAID SEQUENCE DIAGRAM:

```mermaid
sequenceDiagram
    participant User as User Browser
    participant App as Next.js App
    participant API as API Route
    participant DB as Database
    participant Mail as Email Service

    User->>App: Navigate to /auth/register
    App-->>User: Show registration form

    User->>User: Fill email, password, name
    User->>API: POST /api/auth/register
    
    API->>API: Validate input
    API->>API: Hash password with bcryptjs
    
    API->>DB: Create User (emailVerified=null)
    DB-->>API: User created
    
    API->>DB: Create VerificationToken
    DB-->>API: Token created
    
    API->>Mail: Send email with verification link
    Mail-->>User: Email delivered
    
    API-->>User: { success: true, message: "Check your email" }

    Note over User: User clicks email link
    
    User->>API: GET /api/auth/verify?token=xyz
    API->>DB: Find VerificationToken by token
    DB-->>API: Token found
    
    API->>API: Check if token expired
    
    API->>DB: Update User (emailVerified = now())
    DB-->>API: User updated
    
    API->>DB: Delete VerificationToken
    DB-->>API: Token deleted
    
    API-->>User: Redirect to /auth/login
    App-->>User: Show login form
```

TEXT REPRESENTATION:

REGISTRATION FLOW:
  1. User visits /auth/register
  2. User enters email, password, name
  3. Browser sends POST /api/auth/register
  4. API validates input and password strength
  5. API hashes password with bcryptjs
  6. API creates User record (emailVerified = null)
  7. API creates VerificationToken with unique token
  8. API sends email with verification link
  9. User receives email with link
  10. User clicks verification link
  11. Link contains token in URL query param
  12. Browser calls GET /api/auth/verify?token=xyz
  13. API validates token (not expired, exists)
  14. API sets User.emailVerified = now()
  15. API deletes VerificationToken
  16. User redirected to /auth/login
  17. User can now login with email/password

Key Actors:
  • User: Web browser / client
  • RegisterForm: React component
  • API Route (@/app/api/auth/register + verify)
  • Database: Prisma + PostgreSQL
  • Email Service: Nodemailer


3.2 USER LOGIN & SESSION CREATION FLOW
═══════════════════════════════════════

```mermaid
sequenceDiagram
    participant User as User Browser
    participant App as Next.js App
    participant NextAuth as NextAuth
    participant Prisma as Prisma (DB)
    participant JWT as JWT Engine

    User->>App: Navigate to /auth/login
    App-->>User: Show login form

    User->>User: Enter email + password
    User->>NextAuth: POST /api/auth/callback/credentials
    
    NextAuth->>Prisma: Find User by email
    Prisma-->>NextAuth: User record
    
    alt User not found
        NextAuth-->>User: Error: Invalid credentials
    else User found
        NextAuth->>NextAuth: Compare password hash
        
        alt Password incorrect
            NextAuth-->>User: Error: Invalid credentials
        else Password correct
            alt emailVerified is null
                NextAuth-->>User: Error: Email not verified
            else Email verified
                NextAuth->>JWT: Create JWT token
                JWT-->>NextAuth: Token with id, email, role
                
                NextAuth->>Prisma: Update User (sessions)
                Prisma-->>NextAuth: Session created
                
                NextAuth-->>User: Set cookie with JWT
                NextAuth-->>User: Redirect to dashboard
            end
        end
    end
```

TEXT REPRESENTATION:

LOGIN FLOW:
  1. User navigates to /auth/login
  2. User enters email and password
  3. Browser sends credentials to NextAuth callback
  4. NextAuth queries User by email
  5. If user not found → error
  6. If user found, compare password hash
  7. If password incorrect → error
  8. If password correct, check emailVerified
  9. If emailVerified is null → email not verified error
  10. If email verified, create JWT token
  11. JWT contains: id, email, role (from callbacks)
  12. NextAuth creates session in database
  13. NextAuth sets secure HTTP-only cookie
  14. User redirected to authenticated page (dashboard)

Key Actors:
  • User: Web browser
  • LoginForm: React component
  • NextAuth: Authentication library
  • Prisma: Database ORM
  • JWT Engine: Token creation and signing

3.5 CANVAS EDITING FLOW (WITH UNDO/REDO)
═════════════════════════════════════════

```mermaid
sequenceDiagram
    participant User as User
    participant Canvas as Fabric Canvas
    participant Hook as useHistory/Hook
    participant Store as Zustand Store

    Note over User,Store: User draws/edits on canvas

    User->>Canvas: Draw rectangle/text
    Canvas->>Canvas: Add object to canvas
    Canvas->>Hook: Emit object:added event
    
    Hook->>Hook: Serialize current canvas state
    Hook->>Store: addHistorySnapshot(canvasJson)
    
    Store->>Store: Push snapshot to history array
    Store->>Store: Limit to 50 snapshots (FIFO)
    Store->>Store: Clear redo stack
    
    Hook-->>Canvas: Snapshot recorded

    Note over User,Store: User clicks Undo

    User->>Hook: undo()
    
    Hook->>Store: Get previous snapshot from history
    Store-->>Hook: previousCanvasJson
    
    Hook->>Hook: Set isRestoringHistory = true
    Hook->>Canvas: canvas.loadFromJSON(previousCanvasJson)
    Canvas->>Canvas: Recreate objects from JSON
    
    Hook->>Hook: Set isRestoringHistory = false
    Hook->>Store: Push current to redo stack
    
    Note over User,Store: User clicks Redo

    User->>Hook: redo()
    
    Hook->>Store: Get next snapshot from redo stack
    Store-->>Hook: nextCanvasJson
    
    Hook->>Hook: Set isRestoringHistory = true
    Hook->>Canvas: canvas.loadFromJSON(nextCanvasJson)
    
    Hook->>Hook: Set isRestoringHistory = false
```

TEXT REPRESENTATION:

CANVAS EDITING & HISTORY FLOW:
  1. User performs action on canvas (draws, moves, edits)
  2. Fabric.js emits event (object:added, object:modified)
  3. useHistory hook catches event
  4. Hook serializes canvas state with canvas.toJSON()
  5. Hook calls editorStore.addHistorySnapshot()
  6. Zustand appends snapshot to history array
  7. If history exceeds 50 snapshots, remove oldest (FIFO)
  8. Clear redo stack (new action clears forward history)
  9. User can undo multiple times
  10. Click "Undo" button
  11. Hook gets previous snapshot from history
  12. Hook sets isRestoringHistory = true
  13. Hook calls canvas.loadFromJSON(previousJson)
  14. Fabric recreates all objects from JSON
  15. Hook sets isRestoringHistory = false
  16. Previous state restored to redo stack
  17. User can redo to go forward
  18. Click "Redo" button
  19. Hook gets next snapshot from redo stack
  20. Repeat loading process

Key Actors:
  • User: Interacting with canvas UI
  • Fabric.js Canvas: Drawing engine
  • useHistory Hook: Manages undo/redo logic
  • Zustand Store: Persists history snapshots
  • isRestoringHistory Guard: Prevents recursive snapshots
